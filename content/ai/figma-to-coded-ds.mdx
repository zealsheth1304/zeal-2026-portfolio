# Overview

<div className="text-ds-body-1 font-sans text-main mt-8">
    <p className="mb-4"> <span className="font-semibold">Goal:</span> Reducing design-to-dev debt while scaling design systems.</p>
    <p className="mb-4"> <span className="font-semibold">Process:</span> Leveraging agentic AI like Google Antigravity can bridge the gap between design and development. By connecting Antigravity to the Figma MCP, we can automate the extraction of design tokens directly from Figma files into structured JSON. 
    This workflow enables the AI to generate a three-tier token system - primitive, semantic, and component-specific; ensuring scalability. From there, we can prompt Antigravity to "vibe-code" a Storybook-style preview to document states and behaviors, eventually exporting a production-ready Next.js application. </p>
    <p className="mb-4"><span className="font-semibold">Impact:</span> Designers can directly contribute to the production-ready design system, maintain single source of truth and reduce the design-to-dev debt.</p>
    <p className="mb-4 italic text-muted">(I am focusing on using free tools such as Antigravity to ensure this workflow is open to every designer, regardless of budget. You can follow a similar workflow on Cursor or Claude Code too.)</p>
</div>

# Important Tips

<div className="text-ds-body-1 font-sans text-main mt-8">
    <p>To ensure precision, you can implement specific Logic Rules and Skills:</p>
    <ul className="mt-4">
        <li><strong>Logic Enforcement:</strong> If the agent detects a hex code (e.g., #FFFFFF), it must pause and require a token replacement from <code>src/styles/tokens.json</code>.</li>
        <li><strong>Targeted Skills:</strong> When running accessibility checks, the agent can be instructed to exclude specific directories, like <code>/components/ui/</code>, to preserve the visual integrity of core design system components.</li>
        <li><strong>Automated Workflows:</strong> A "Ship Component" workflow can automate the entire lifecycle, from token extraction to final code.</li>
    </ul>
</div>

# Step-by-Step Process

<div className="text-ds-body-1 font-sans text-main mt-8">
    <div className="flex flex-col gap-8 items-start bg-slate-50 dark:bg-slate-300/10 p-8 rounded-2xl border border-border-subtle px-8 py-12 my-12">
        <p className="font-semibold"> Step 1: Create Design Tokens in Figma</p>
        <p> Run free Figma Plugin 'Tokens Studio for Figma' to Export Styles and Colors into tokens.json file</p>
        <CaseStudyImage src="/assets/AI/1_FigmatoCodedDS/DesignTokens.webp" label="Figma Tokens" className="mt-8 pb-8 max-w-sm" />
        <p className="font-medium"> Reduce Design-to-Dev Debt by organizing tokens into 3 levels:</p>
        <ul className="list-disc pl-6 space-y-2">
            <li><strong>Global Tokens:</strong> Raw values (e.g., `blue-500: #3b82f6`).</li>
            <li><strong>Alias/Semantic Tokens:</strong> How the color is used (e.g., `brand-primary: {colors.blue-500}`).</li>
            <li><strong>Component Tokens:</strong> (Optional) Specific to a component (e.g., `button-bg: {brand-primary}`).</li>
        </ul>
        <p className="mb-4"> Use the "+" icon in the plugin and using curly braces `{}` to reference other tokens.</p>
    </div>
