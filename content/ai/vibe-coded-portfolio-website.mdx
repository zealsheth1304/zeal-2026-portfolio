    <div className="flex flex-col gap-8 items-start bg-slate-50 dark:bg-slate-300/10 p-8 rounded-2xl border border-border-subtle px-8 py-12 my-12">
        <p className="mb-4 font-semibold"> Step 3: Use SEO-Optimized best practices to organize the website content </p>
        <p><strong> SEO Best Practices for .mdx Frontmatter: </strong></p>
        <ul className="list-disc pl-6 space-y-2">
            <li><strong>`title`:</strong> Compelling and keyword-rich (50-60 characters).</li>
            <li><strong>`description`:</strong> Concise, keyword-rich summary (150-160 characters) that encourages clicks.</li>
            <li><strong>`keywords`:</strong> While less critical for direct ranking, still good for contextual understanding.</li>
            <li><strong>`slug`:</strong> Clean, descriptive, and containing relevant keywords.</li>
            <li><strong>`coverImage` & `altText`:</strong> Essential for social sharing (Open Graph) and image SEO. Use descriptive `altText`.</li>
            <li><strong>Structured Data (Schema.org):</strong> Consider adding JSON-LD schema for `Article` or `WebPage` to your page templates to help search engines understand your content better. Tools like Schema Markup Generator can help.</li>
        </ul>
        <p><strong> Image Optimization: </strong></p>
        <ul className="list-disc pl-6 space-y-2">
            <li><strong>Modern Formats:</strong> Always use `webp` for images and `webm` for videos. These offer superior compression with minimal quality loss compared to JPG/PNG/MP4.</li>
            <li><strong>Responsive Images:</strong> Use `srcset` and `<picture>` tags to deliver appropriately sized images based on the user's viewport.</li>
            <li><strong>Lazy Loading:</strong> Implement `loading="lazy"` on images and iframes to defer loading of off-screen content.</li>
            <li><strong>CDN (Content Delivery Network):</strong> Serve static assets (images, CSS, JS) from a CDN to reduce latency for users worldwide.</li>
        </ul>
        <p><strong> Font Optimization: </strong></p>
        <ul className="list-disc pl-6 space-y-2">
            <li><strong>Variable Fonts:</strong> Use variable fonts to reduce font file size.</li>
            <li><strong>Self-Host Fonts:</strong> Host Google Fonts or other fonts yourself to avoid extra DNS lookups.</li>
        </ul>
        <p><strong> Server-Side Rendering (SSR) / Static Site Generation (SSG): </strong></p>
        <ul className="list-disc pl-6 space-y-2">
            <li>For portfolio sites, SSG (e.g., with Next.js, Astro, SvelteKit) is often ideal, as it generates pre-built HTML files, leading to incredibly fast initial loads.</li>
        </ul>
